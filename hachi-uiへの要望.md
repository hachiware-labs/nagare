# hachi-uiへの要望

## 背景

Nagare のランタイム設定画面を作る中で、設定画面では「表示できる情報を全部出す」よりも、「ユーザーが次の判断に使う情報だけを残す」ことが重要だと分かった。

特に、ランタイムやモデル選択のような管理画面では、実装上は正しい情報でも、ユーザーの選択や実行に効かない説明・ラベルが画面を重くする。

## hachi-uiに取り入れたいこと

### 1. 設定画面では、実装由来の説明を初期表示しない

次のような情報は、開発者には意味があるが、ユーザーの判断には効きにくい。

- `PATH上のコマンドを使用`
- `models listから取得`
- `visibility=list`
- `provider/model形式`
- `追加入力なし`

必要ならヘルプ、詳細、診断、開発者向け表示に下げる。

### 2. 色付きラベルは状態・警告・差分に限定する

残すべきラベル:

- `未実行`
- `実行済み`
- `失敗`
- `承認待ち`
- `未接続`

削るべきラベル:

- `モデル一覧取得可`
- `一覧取得なし`
- `アプリ既定`
- `選択中`
- `追加入力なし`

色は「状態変化」「警告」「人間が判断すべき差分」にだけ使う。

### 3. 未検出のものは選択肢に出さない

ドロップダウンには、ユーザーが今選べるものだけを表示する。

- Runtime候補には検出済みRuntimeだけを出す
- 未検出Runtimeは診断画面、接続画面、空状態、エラー表示で扱う
- 選択できないものを薄く出すより、主導線から外した方が分かりやすい

### 4. 上位選択を下位項目で繰り返さない

Providerを別のドロップダウンで選んでいるなら、モデル欄でProvider名を繰り返さない。

例:

- 悪い: `provider/model を選択`
- 良い: Providerを選んだ後、モデル欄にはそのProviderのモデル名だけを出す

### 5. 必要になった時だけ入力欄を出す

条件付きの入力欄は、必要な条件が選ばれるまで隠す。

例:

- Claude Code: モデル一覧が取れないので、`既定モデルを使う` / `モデル名を手入力する`
- OpenClaw: ProviderがOpenAIならBase URLを出さない
- OpenClaw: ProviderがOllama / LM StudioならBase URLを出す

### 6. フォームは「作業を進めるための情報」に絞る

初期表示で残す情報:

- 何を選ぶか
- 何を入力するか
- 現在の実行・接続状態は何か
- 次に押すボタンは何か

初期表示から下げる情報:

- どう取得したか
- どの内部コマンドを使うか
- なぜ候補が作られたか
- 既定値が内部的にどう決まるか

## hachi-uiへの反映方針

SKILL.md本体には短いトリガーだけ入れる。

詳細な判断ルールは `references/form-settings-ui-rules.md` に切り出し、フォーム、設定画面、管理画面、初期セットアップ画面を作る時だけ読む。

これにより、hachi-ui全体を重くせず、今回の学びを必要な場面で再利用できる。
